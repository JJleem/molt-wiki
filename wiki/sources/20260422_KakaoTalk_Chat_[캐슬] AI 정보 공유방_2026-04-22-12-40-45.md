## 핵심 인사이트

### 모델 선택 및 비교
- **Claude**: 맥락 파악/기획에 우수하나 한글 토큰 효율 낮음, Opus 4.6/4.7은 이미지 분석 및 코딩 성능 향상
- **Codex**: 지시사항 정확 수행, 토큰 가성비 우수, 5월 초 토큰 2배 이벤트 진행
- **GPT-5.4/5.5**: 한글 토큰 효율 개선, 빠른 업데이트 속도
- **Gemini**: 토큰 제한 거의 없음, 가성비 최고지만 코딩 성능 약함
- **로컬 모델**: Qwen 2.5 Coder 7B가 코딩 최우수, M1 Max 64GB 이상 권장

### 토큰 효율 관리
- 새 세션마다 SESSION_LOG.md에 작업 내용/남은 작업/우선순위 기록
- 컴팩트 대신 세션 종료 후 인수인계 방식으로 연속성 유지
- 한글은 영어 대비 2-3배 토큰 소비 → 영어로 사고하고 최종 답변만 한글화
- Claude 프롬프트 캐싱 활용 시 약 90% 토큰 할인 (5분 유지)
- /btw, /compact, /clear 명령어로 토큰 절약

### 요금제 및 구독 전략
- Claude Pro($20): 토큰 제한적, Sonnet만으로도 주간 부족
- Claude Max($110): Pro 대비 체감 차이 크며 실무 필수
- Claude Code($200): 개발 최적 구독, Cursor($20) 대비 성능 우수
- 구독제가 API 종량제보다 4-6배 비용 효율
- 다중 계정으로 비용 최적화 가능

### AI 코딩 도구
- **Claude Code**: 로컬 작업 직접 확인 가능, VS Code 통합, Hooks로 Git 자동화
- **Cursor**: 가성비 우수(20달러), Composer 2는 140페이지에 14% 토큰만 소비
- **Codex**: 병렬 활용으로 코드 검증 강화
- Windows 환경보다 WSL/Linux 기반에서 실행 속도 향상
- Desktop App은 터미널 CLI 대비 토큰 1.2-2배 소비

### 에이전트 설계 원칙
- 기획에 70% 이상 시간 투자, 구조화된 아키텍처 우선
- 서브에이전트보다는 문서화와 스킬 분리 권장
- 병렬 처리는 자연스럽게 필요할 때 구현
- 계획 담당 에이전트와 실행 에이전트 분리로 토큰 절약
- AGENTS.md는 200줄 미만 권장

### 코드 리뷰 전략
- 비싼 모델(Claude)과 저렴한 모델(Codex) 병렬 활용
- 최종적으로 개발자가 문제 해결 여부 판단
- 코드의 단순성과 이해가능성 평가 필수
- Verification Agent로 검증 단계 추가

### 프롬프트 엔지니어링
- 페르소나 설정보다는 출력값 형식 지정이 토큰 효율적
