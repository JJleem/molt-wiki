# LLM 모델 선택 전략 — Haiku vs Sonnet

동일한 Claude 계열에서 작업 성격에 따라 모델을 분리해 비용 대비 품질을 최적화하는 전략.

## 분리 기준

| 작업 유형 | 적합한 모델 | 이유 |
|-----------|------------|------|
| 판단 / 선별 / 종합 | Sonnet | 복잡한 추론, 맥락 통합 능력 필요 |
| 패턴 인식 / 분류 / 반복 작업 | Haiku | 고성능 불필요, 속도·비용 우선 |
| 멀티모달 (Vision) | Sonnet | 이미지 이해 품질 차이가 결과에 직접 영향 |
| 앙상블 오케스트레이터 | Sonnet | 다수 결과를 종합·선별하는 최종 판단 역할 |

## 적용 예시 (Centinel 파이프라인)
- Haiku: 장르별 인기 요인 분석(배치), 앙상블 소재 생성(A/B 병렬)
- Sonnet: 트렌드 분석, Vision 스크린샷 분석, 최종 소재 선별 오케스트레이터

## 핵심 원칙
같은 모델이어도 시스템 프롬프트를 다르게 주면 전혀 다른 방향의 결과가 나온다.
Haiku × 2 병렬(분석형 A + 창의형 B)은 비용은 낮게, 다양성은 넓게 확보하는 패턴.
