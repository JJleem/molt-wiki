# 토큰 효율 최적화

## 세션 연속성 관리

- 새 세션마다 `SESSION_LOG.md`에 작업 내용 / 남은 작업 / 우선순위 기록
- `/compact` 대신 세션 종료 후 **인수인계 방식**으로 연속성 유지
- `/btw`, `/compact`, `/clear` 명령어로 토큰 절약

## 언어별 토큰 소비

- 한글은 영어 대비 **2-3배 토큰 소비**
- 실용 전략: 영어로 사고 → 최종 답변만 한글화

## Claude 프롬프트 캐싱

- 캐싱 활용 시 약 **90% 토큰 할인** (5분간 캐시 유지)
- 동일한 시스템 프롬프트/문서를 반복 사용하는 경우 효과 극대화

## 환경별 토큰 소비 차이

- Desktop App은 터미널 CLI 대비 **1.2-2배 토큰 소비**
- Windows 환경보다 WSL/Linux 기반에서 실행 효율 향상

## 비용 비교

- 구독제가 API 종량제보다 **4-6배 비용 효율** 높음
- 비싼 모델(Claude) + 저렴한 모델(Codex) 병렬 활용으로 비용 절감
