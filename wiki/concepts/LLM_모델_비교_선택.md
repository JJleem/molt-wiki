# LLM 모델 비교 및 선택 전략

## 주요 모델 특성

| 모델 | 강점 | 약점 | 비고 |
|------|------|------|------|
| **Claude** | 맥락 파악, 기획, 코딩 성능 | 한글 토큰 효율 낮음 | Opus 4.6/4.7은 이미지 분석 향상 |
| **Codex** | 지시사항 정확 수행, 토큰 가성비 | - | 5월 초 토큰 2배 이벤트 |
| **GPT-5.4/5.5** | 한글 토큰 효율, 빠른 업데이트 | - | |
| **Gemini** | 토큰 제한 거의 없음, 가성비 최고 | 코딩 성능 약함 | |
| **로컬 모델** | 비용 없음, 프라이버시 | 하드웨어 요구 높음 | Qwen 2.5 Coder 7B 코딩 최우수 |

## 로컬 모델 실행 요구사항

- Qwen 2.5 Coder 7B: 코딩 작업에서 로컬 모델 최우수
- M1 Max 64GB 이상 권장

## 구독 요금제 전략

- **Claude Pro ($20)**: 토큰 제한적, Sonnet만으로도 주간 부족
- **Claude Max ($110)**: Pro 대비 체감 차이 크며 실무 필수
- **Claude Code ($200)**: 개발 최적 구독, Cursor($20) 대비 성능 우수
- 구독제가 API 종량제보다 **4-6배 비용 효율** 높음
- 다중 계정으로 비용 최적화 가능

## 선택 가이드

- 기획/맥락 이해 중심 → Claude
- 코딩 지시 정확 수행 + 비용 절감 → Codex
- 긴 컨텍스트 필요 → Gemini
- 비용 없이 코딩 → Qwen 2.5 Coder 7B (로컬)
- 비싼 모델(Claude) + 저렴한 모델(Codex) **병렬 활용**으로 코드 리뷰 강화
